Unit 1

Expressions

Are the fundamentals of programming. These make up the most primitive forms of data that we will be learning how to manipulate and control to write code.

Variables

Variables allow us to create meaning the data that we will be controlling. Creating expressions will not accomplish much if we do not assign meaning to these expressions. This process of assigning meaning to data is called data abstraction.

Operators

Operators come in many shapes and forms in programming. You should already be familiar with the most commonly used ones from your math classes. Operators are items such as plus, minus, multiply and divide. In programming we will learn several new ones that we can use to write programs.


Unit 2

Control Flow

This is how we can control the flow of information. We will be dealing with a large amount of information and know how to control it is important.

Conditional Statements

The first type of control flow we will learn is conditional statements. This is more commonly known as if statements. We check the condition and if it the condition is true we take one path as apposed to another.

Iterations

The next main form of flow of information will be done using iterations. Iterations are more commonly called loops. They set up a block of code that we will repeat until a certain condition is met.


Unit 3

Functions

Functions in programming allow us to create our own forms of data and use it. This is the next step of data abstraction. Creating good functions requires practice and patience. Designing an efficient algorithm can speed up your program drastically.

Arrays

Arrays are an important type of data that gets its own section. We will learn about several built in data types in java but spend more time on learning how to use arrays. Essentially they are a data type that can hold several different data types.

Debugging

Using the compilers debugger is a great too to learn how to debug your code. You will without doubt have your code give one output when you were expecting another output. Using the debugger is a much easier way to find the solution of why your output was not as expected.


Unit 4

Classes

Classes re the back bone of Java. Everything is programmed in a class. This makes java a great Object Orientated Programming. In this we will look at the basic properties of a class and how to set them up.

Using a Class

In this section we will look at how we can create an instance of a class and use its features. This is when your programming will really start to open up options of what you can create.

ArrayList Class

A huge part of being a good programmer is finding pre-built classes and libraries that already perform functions. Now that we are understanding classes we can start using these in our programs. The first one we will explore is the ArrayList Class.


Unit 5

Data Abstraction

This introduces tests driven ways of designing your data abstraction. Using this approach at first may seem more complicated than just coding something in however, once programs become larger testing is more difficult.

4 Phases of Data Abstraction

The 4 phases of data abstraction are:

  • Specification

  • Usage Scenarios

  • Testing

  • Implementation

Using this process writing the code is the 4th and final step.


Unit 6

GUI's

Almost all programs now use Graphical User Interfaces to make interacting with the program easier. We will learn the basics of creating GUI's and apply what we have learned so far to create programs.

Primitive vs Reference Data

It is time to have a deeper understanding of the different data "types" that we use in programming. Understanding how these behave differently is necessary for effective programming.
