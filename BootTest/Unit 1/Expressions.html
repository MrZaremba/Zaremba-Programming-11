Expressions Part 1

Expressions are used in programming to evaluate primary data. Expressions in programming are much like expressions in math. We will have at least 2 features to an expression, they must have Operators and Operands

Operands are what we call primitive forms of data. We will be working with 4 main types

  • integers, numbers without decimals

  • floats, which are numbers with decimals

  • booleans, true or false

  • characters, single letters

There are many operators but first we will look into mathematical operators. We will primarily be using these 7 throughout the course.

  • addition '+'

  • subtraction '-'

  • multiplication '*'

  • division '/'

  • modulus '%'

  • increment '++'

  • decrement '--'

To build a basic expression we have <operand><operator><operand>. An example would be 5 + 6

Evaluation

Evaluating an expression is how the computer reads the expression and assigns a value to it. In the example above 5 + 6 = 11

This may seem obvious but later on how expressions evaluate can become confusing. It is also important to understand that the compiler does not care about the 5 + 6 part is simply reads it as 11. Programming language like math also rely on Order of operations.

Something is wrong

Inside of your System.out.println() try running the expression '28 / 6' and see what you get. Notice that outcome is simply 4. This is because 28 and 6 are both integers. The compiler reads the data as integers which means no decimal places. So when it evaluates this expression is only reads the integer value. It ignores any fraction that is left over. It is not rounding the number it is simply Cutting off the value of the fraction
