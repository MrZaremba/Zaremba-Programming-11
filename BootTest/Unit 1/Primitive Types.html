Primitive Data Types

Primitive data types will be the first type of data in we will learn to store inside of our programs. To store information we need to create what we call variables. Variables in java either point to primitive data stored on the on the computer or they will point to a reference to an object (more on this later).

Variables in programming are very similar to variables in math. They are place holders for information we do not yet have. These variables can change as the program is running. To create a variable in java we must declare a Type. The table below is a list of the most commonly used primitive types:

Type Abbreviation Description
Integer int Positive and negative whole numbers to a maximum or minimum value
Double double Numbers that contain decimals. 2 pieces of information is stored. Value to either side of decimal
Boolean boolean True or False value
Character char A single letter

Declaring a Variable

To declare a variable we first declare the type then the name followed by a semicolon <type> <name>;
Example: int x;
This creates a spot in memory and names that location 'x'. After we have declared the variable we can use an assignment operator '=' to specify what integer value you wish it to hold
x = 10;

We can now use the variable we create in code to evaluate expressions. Look at the following example.
int x = 10;
int y = x * 2;
In this example we first declare and assign in one line the variable 'x' and assign it 10. Then we created and assigned the variable 'y' and used the value from 'x' as a number in the expression.

Data Abstraction

The concept of data abstraction will be looked at over the entire course. Data abstraction is the idea of assigning meaning to the information that is stored in the program. For primitive types of data it is fairly simplistic. Once we start learning how to code more complex data the abstraction can become difficult. Here is a simple way we can assign meaning to a primitive data type.
boolean lightsOn = true;
This even to someone who knows little to nothing about code can abstract what information this variable is holding. By assigning appropriate names to values we can give the values meaning.
