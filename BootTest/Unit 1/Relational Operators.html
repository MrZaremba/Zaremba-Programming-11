Relational Operators

Recall the operators from our previous lesson in expressions:

  • addition '+'

  • subtraction '-'

  • multiplication '*'

  • division '/'

  • modulus '%'

  • increment '++'

  • decrement '--'

We have additional operation we can perform called relational operators. These are used to compare 2 values. When relational operators evaluate they will always result in a boolean (true or false). The table below shows what relational operators we will be using (Not all of them are listed just the commonly used operators):

Operator Description
< Tests if a value is less than another value
> Tests if a value is greater than another value
<= Tests if a value is less than or equal to another
>= Tests if a value is greater than or equal to another
== Tests if a value equal to another
!= Tests if a value is not equal to another

These operations are for testing primitive data only. We can not use them on testing other forms of data. Currently the only none primitive form of data we can make are Strings. We will learn others and the same rule will apply. It is also important to note comaparing equality is not done with an '=' sign but with '=='. This is because '=' is call an assignment operator. It assigns values to variables. We need to use something else to compare equality which is why we use '=='.

Evaluating Relational Operators

Remember that when we evaluate comaparative operators we will get a boolean as a result. For example, 10 > 9 is true while 10 < 9 is false. Examples in code:
System.out.println(10 != 10);
evaluates to false. This is because 10 is equal to 10 which means the statement 10 ! 10 is false.
